首先创建一个简单的日志类，文章中都使用它来在控制台上输出：

import java.util.Calendar;

public class MyLogger {
    public void info(String message) {
        Calendar calendar = Calendar.getInstance();
        String time = ""+calendar.get(Calendar.HOUR_OF_DAY)+":"+calendar.get(Calendar.MINUTE)+":"+
            calendar.get(Calendar.SECOND)+"."+calendar.get(Calendar.MILLISECOND);
        String threadName = Thread.currentThread().getName();
        System.out.println(time+" ["+threadName+"] - "+message);
    }
}

1 创建线程

1.1 继承 Thread，重写 run 方法

Thread t = new Thread("t") {
    @Override
    public void run() {
        log.info("thread running");
    }
};
t.start();

这里我们创建一个匿名内部类来继承 Thread 类，run() 方法中定义子线程的执行逻辑，start() 方法启动线程（转为就绪状态，不一定立即运行），结果如下：

22:27:38.504 [t] - thread is running

1.2 实现 Runnable 接口

由于 Runnable 是一个函数式接口，所以可以使用 Lambda 表达式：

Thread t = new Thread(() -> {
    log.info("thread is running");
}, "t");
t.start();

其原理与上一个方法相同的，此时子线程执行的是 Runnable 中的 run() 方法，详见 Thread 类中 run() 方法的实现。结果如下：

22:29:2.925 [t] - thread is running

1.3 使用 Callable 和 FutureTask

子线程可以返回一个值，并能在主线程获取到它。

// 参数是一个实现了 Callable 接口的对象
FutureTask<Integer> task = new FutureTask<>(() -> {
    log.info("thread is running");
    Thread.sleep(2000);
    return 100;
});
new Thread(task, "t").start();

// get 方法将阻塞至 FutureTask 执行的方法返回
Integer result = task.get();
log.info("FutureTask 返回结果："+result);

22:33:53.746 [t] - thread is running
22:33:55.769 [main] - FutureTask 返回结果：100

如上所示，主线程阻塞 2s 直至 FutureTask 执行的方法返回。

2 Thread 类常用方法

Thread.sleep(2000) 睡眠指定毫秒。
如果此时当前线程的打断标记为 true，则不会睡眠；或者睡眠过程中被打断，则线程将被唤醒进入就绪状态。并抛出 InterruptedException，同时将打断标记设为 false。
t.join()、t.join(1000) 阻塞当前线程，直到 t 线程执行完毕，或者设置一个最大的等待时间（ms）。
如果此时 t 线程打断标记为 true，则不会等待；或者等待过程中被打断，则线程将被唤醒进入就绪状态。并抛出 InterruptedException，同时将打断标记设为 false。
t.interrupt() 打断 t 线程中的一些等待方法，比如 Thread.sleep()、t.join()、obj.wait() 等，设置 t 线程的打断标记为 true。见上面两个方法。
t.isInterrupted() 获取 t 线程的打断标记的值。
Thread.interrupt() 获取 t 线程的打断标记的值，同时将其设为 false。
Thread.currentThread() 获取当前线程。
t.getState() 获取 t 线程的状态。
Thread.yield() 让当前线程进入就绪状态，重新竞争 cpu。但是该方法并不可靠，它实际上只能起到一个提示作用，因此业务逻辑不要依赖于这个方法。
t.setPriority(1) 设置 t 线程的优先级，优先级的范围为 1~10，优先级越高，越容易被 cpu 调度执行。
但是该方法并不可靠，它实际上只能起到一个提示作用，最终优先级还是由操作系统决定，因此业务逻辑不要依赖于这个方法。
t.isAlive() 线程是否存活（是否还没执行完毕）。

interrupt() 方法示例：

Thread t = new Thread(() -> {
    try {
        Thread.sleep(1000); // 2.方法被打断，抛出 InterruptedException，并将打断标记设为 false
    } catch (InterruptedException e) {
        log.info("sleep 被打断，打断标记："+Thread.currentThread().isInterrupted());
    }
}, "t");
t.start();

Thread.sleep(50);

t.interrupt(); // 1.打断线程 t 的 sleep() 方法，并设置打断标记为 true

23:58:7.809 [t] - sleep 被打断，打断标记：false

3 线程的状态

Java 中的线程状态与操作系统中的线程状态不同，一共有 6 种状态，定义在 Thread.State 枚举类中：

  • NEW：Thread 对象已经创建，但还没有调用 start() 方法。
  • RUNNABLE：线程处于就绪、运行、阻塞状态（比如 IO 阻塞）。
  • WAITING：线程内调用了 obj.wait()、t.join()、LockSupport.park() 等方法主动等待，而处于阻塞状态。
  • TIMED_WAITING：线程内调用了 obj.wait(n)、Thread.sleep(n)、t.join(n) 等超时等待的方法，而处于阻塞状态。
  • BLOCKED：线程竞争锁失败而阻塞，等锁被释放后，线程将被唤醒重新竞争锁。
  • TERMINATED：线程终止。


可以通过 getState() 方法来获取线程的状态：

Thread t = new Thread(...);
t.getState(); // 获取 t 线程状态

4 守护线程

默认情况下，Java 进程需要等待所有线程都运行结束，才会结束。
可以设置线程为守护线程，只要其它非守护线程运行结束了，即使守护线程的代码没有执行完，也会强制结束。

Thread t = new Thread(...);
t.setDaemon(true); // 设置 t 线程为守护线程

5 应用

5.1 Thread.sleep() 降低 cpu 占用

考虑这种情况，我们需要在服务器中执行一个循环，监听客户端发送的请求，但是当没有请求时，循环一直执行，占用 CPU，但是却什么都没干，这时就可以使用 Thread.sleep() 来降低 CPU 占用。

while(true) {
    try {
        Thread.sleep(50);
    } catch (InterruptedException e) {
        e.printStackTrace();
    }
    // 处理请求...
}

5.2 两阶段终止模式

如果我们想要在线程 t1 中终止线程 t2，不应该由 t1 强制终止 t2，比如执行在 t1 中执行 t2.stop() 方法，因为这样的话，如果 t2 占用了锁，那么会导致锁得不到释放。
正确的方式是给 t2 传递一个信息，让 t2 自己去释放资源，自己终止。
可以在 t1 线程中调用 t2.interrupt() 方法设置打断标记，t2 线程检查打断标记后，自己终止：

Thread t = new Thread() {
    @Override
    public void run() {
        while(true) {
            Thread current = Thread.currentThread();
            if(current.isInterrupted()) { // 检查打断标记
                break;
            }
            try {
                Thread.sleep(50); // 降低 CPU 占用
            } catch (InterruptedException e) {
                current.interrupt(); // Thread.sleep() 会将打断标记设为 false，所以这里需要重新设置
                e.printStackTrace();
            }
            // 处理请求...
        }
    }
};
// 如果需要终止 t 线程，只需调用 t.interrupt()
